# Changelog

## [5.1.0] - 2026-06-14

### Added
- RU/EN переключатель языка в попапе. Выбор сохраняется в `chrome.storage.local`
- `?lang=en` для виджета — текст заглушки на английском
- Источник медиа (hostname вкладки) отображается в одной строке с заголовком «Сейчас играет»
- Таймер обратного отсчёта в статусе отключения («• повтор через Nс»)
- Автоматическое переподключение при перезапуске OBS (сердцебиение 15 секунд вместо 60)
- Обнаружение включённой аутентификации в OBS — расширение показывает ошибку и просит снять галочку
- `chrome.tabs.onRemoved` — очистка трека при закрытии вкладки
- Страховочная чистка треков: 65 секунд без обновления → удаление
- `nullCount` в content.js: 3 пустых опроса подряд → `trackEnded`
- Поддержка Яндекс.Музыки: чтение currentTime через `valueAsNumber`

### Fixed
- Текст «Ожидание подключения...» теперь использует `.no-music-text` (внутренний элемент), а не `#no-music` — заглушка получает правильные размеры для h/v/compact
- Виджет перерисовывается при повторных данных с тем же треком после watchdog
- `wsConnecting` в `onerror` теперь проверяет `ws !== sock` перед сбросом флага
- Прогресс-бар в Яндекс.Музыке реагирует на скролл (исправлен `getAttribute('value')` на `valueAsNumber`)

### Removed
- Полностью удалена аутентификация: `computeAuth()`, `wsPassword`, поле пароля из DOM и CSS, все закомментированные блоки
- Удалены серверные скрипты: `launch.vbs`, `server.js`, `restart.bat`, `stop.bat`
- Удалён отладочный вывод `[wsState/wsAuth/wsConn] port:` из попапа

### Changed
- Попап переведён на русский: шаги настройки OBS, сообщения об ошибках
- `onerror` порядок: проверка `ws !== sock` теперь до сброса `wsConnecting`
- README.md обновлён под текущую архитектуру
- Описание расширения в manifest.json на русском
